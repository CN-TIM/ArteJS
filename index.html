Overview

A powerful, extensible, flexible, cross-browser rich text editor.

ArteJS is an integral part of VistaPrint's DIY product design experience that allows adding rich text to a document that eventually gets printed. This imposes a stringent requirement on the output of the rich text editor. Most of the rich text editors we evaluated either did not allow any control on the resultant markup or were simply buggy/incomplete.

In addition, ArteJS provide many other features that are expected of a modern rich text editor library.

  • A simple and intuitive jQuery like API.
    For example: 
    $(".editor").Arte(); // Turns .editor into rich text editor 
$(".editor").bold(); // Make the selected text bold
$(".editor").fontSize(10); // Change fontsize fo selected text to 10px
  • A small dependency footprint: jQuery and Rangy 1.2.3 (Included with ArteJS).
  • Full control over the resultant HTML : TODO: Links to examples with Classes/Styles/Tags for a command
  • A loosely coupled architecture based on a plugins that can be used overwrite any existing or add new functionality. In fact, a large part of the functionality is built using plugins.
  • ArteJS supports a clean interface for toolbar integration. TODO: Link to reference toolbar implementations that can support multiple editors with a single toolbar or one toolbar for each editor.
  • Semantically concise and valid markup: e.g. no redundant styles, classes or tags.
  • A rich event model provide a rich integration point. TODO: Link to supported events
  • Keyboard shortcut integration
  • State detection so that it's easy to figure out the state of selected html.
  • Highly granular configuration.
  • Copy/paste functionality from Microsoft Word.
  • Open sourced so it's free--ArteJS is available on an Apache2 license.
  • Most importantly, ArteJS is used in production millions of times a day

Getting Started

Get ArteJS

Download the latest release of ArteJS

Typical page structure structure

        <!-- Markup to host the toolbar and the editor-->
        <div class="toolbar"></div>
        <div class="editor"></div>
        <!--Included the Rich text files-->

        <!--Add dependencies and ArteJS Scripts-->
        <script src="../path/to/jquery-1.9.1.js"></script>
        <script src="../path/to/ArteJS/rangy.debug.js"></script>
        <script src="../path/to/ArteJS/arte.debug.js"></script>
        <script src="../path/to/ArteJS/toolbar.debug.js"></script>

        <script type="text/javascript">
            $(function () {
                // List and sequence of the toolbar buttons 
                var buttons = ["bold", "italic", "underline", "blockquote", "textAlignLeft", "textAlignCenter", "textAlignRight", "h1", "h2", "h3", "h4", "h5", "h6", "subscript", "superscript", "unorderedList", "orderedList", "undo", "redo", "fontSize", "fontFamily", "color", "backgroundColor", "insertLink"];

                // Initialize the Arte editor with default options
                $(".editor").Arte();

                // Initialize the toolbar with default options
                $(".toolbar").ArteToolbar({
                    buttons: buttons
                });
            });
        </script>

Rich Text API Reference

Supported rich text commands

A rich text command can be applied using a html tag, a style or a class name. ArteJS allows applying the command in either of these three mechanisms.
Note that the default configuration excluded the use of deprecated html tags (e.g.: font). Configuration can be easily updated to used the deprecated html tags if needed.

Commands that can be applied using Tag/Style/Class

  • Bold
  • Italic
  • UnderLine

Commands that can be applied using Style/Class

  • Alignment
  • Background color
  • Font size
  • Font family
  • Font color

Commands that can be applied using a html Tag

  • Blockquote
  • H1-H6
  • Subscript/Superscript
  • UL/OL

Other commands

  • Insert Link (Implemented using a plugin)

Compiling ArteJS

ArteJS uses Grunt to minify, combine, analyze code and run unit tests. Following are the steps to run grunt tasks:
  • Install Node
  • Install Github client
  • Create and navigate to a local directory for ArteJS code
  • Clone the git repository
  • Install node dependencies: execute "npm install"
  • Build ArteJS: Execute "Grunt all"
There are many other grunt tasks defined for conveniance.
